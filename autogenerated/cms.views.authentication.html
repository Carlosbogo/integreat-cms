Authentication¶

This package contains views related to login, logout and password reset functionality

Authentication Actions¶

class cms.views.authentication.authentication_actions.MfaEnableAuthentication(**kwargs)[source]¶

Bases: django.contrib.auth.views.LoginView

View to extend the default login behaviour from LoginView with multi-factor-authentication.

form_valid(form)[source]¶

This function overwrites form_valid() which is called if the login form is valid. In case the user has mfa-keys configured, the login is delegated to mfa(). Else, the default method login() is used to log the user in. After that, the user is redirected to LOGIN_REDIRECT_URL.

Parameters

form (AuthenticationForm) – User login form

Returns

Redirect user to mfa login view or to LOGIN_REDIRECT_URL

Return type

HttpResponseRedirect

cms.views.authentication.authentication_actions.mfaAssert(request)[source]¶

Generate challenge for multi factor authentication. If the user did not provide the first factor (password) or already authenticated with multiple factors, an error is returned. This AJAX view is called asynchronously by JavaScript.

Parameters
