Languages¶

This package contains both language-related data models: Language and LanguageTreeNode

Language¶

class cms.models.languages.language.Language(*args, **kwargs)[source]¶

Bases: django.db.models.base.Model

Data model representing a content language.

Parameters:
  • id (django.db.models.AutoField) – The database id of the language
  • code (django.db.models.CharField) – The bcp47 code of the language (see RFC 5646). The recommended minimum buffer is 35 (see Section 4.4.1.). It’s unlikely that we have language codes longer than 8 characters though.
  • native_name (django.db.models.CharField) – The native name of the language
  • english_name (django.db.models.CharField) – The name of the language in English
  • text_direction (django.db.models.CharField) – The text direction of the language (choices: cms.constants.text_directions)
  • created_date (django.db.models.DateTimeField) – The date and time when the language was created
  • last_updated (django.db.models.DateTimeField) – The date and time when the language was last updated

Reverse relationships:

Parameters:
  • language_tree_nodes (Reverse ManyToOne Relation from LanguageTreeNode) – All language tree nodes of this language
  • page_translations (Reverse ManyToOne Relation from PageTranslation) – All page translations in this language
  • event_translations (Reverse ManyToOne Relation from EventTranslation) – All event translations in this language
  • poi_translations (Reverse ManyToOne Relation from POITranslation) – All poi translations in this language
  • push_notification_translations (Reverse ManyToOne Relation from PushNotificationTranslation) – All push notification translations in this language

Language Tree Node¶

class cms.models.languages.language_tree_node.LanguageTreeNode(*args, **kwargs)[source]¶

Bases: mptt.models.MPTTModel

Data model representing a region’s language tree. Each tree node is a single object instance and the whole tree is identified by the root node. The base functionality inherits from the package django-mptt (Modified Preorder Tree Traversal).

Parameters:
  • id (django.db.models.AutoField) – The database id of the language tree node
  • active (django.db.models.BooleanField) – Whether or not it should be possible to create new pages in this language and content of this language should be delivered via the API
  • created_date (django.db.models.DateTimeField) – The date and time when the language tree node was created
  • last_updated (django.db.models.DateTimeField) – The date and time when the language tree node was last updated

Fields inherited from the MPTT model (see Models and Managers for more information):

Parameters:
  • tree_id (django.db.models.PositiveIntegerField) – The id of this language tree (all nodes of one language tree share this id)
  • lft (django.db.models.PositiveIntegerField) – The left neighbour of this node
  • rght (django.db.models.PositiveIntegerField) – The right neighbour of this node
  • level (django.db.models.PositiveIntegerField) – The depth of the node. Root nodes are level 0, their immediate children are level 1, their immediate children are level 2 and so on…

Relationship fields:

Parameters:
  • parent (TreeForeignKey to LanguageTreeNode) – The parent node of this node (related name: children)
  • language (ForeignKey to Language) – Language this tree node refers to (related name: language_tree_nodes)
  • region (ForeignKey to Region) – The region this node belongs to (related name: language_tree_nodes)

Reverse relationships:

Parameters:children (Reverse ManyToOne Relation from LanguageTreeNode) – The children of this language tree node

get_ancestors(*args, **kwargs)[source]¶

Creates a QuerySet containing the ancestors of this model instance.

This defaults to being in descending order (root ancestor first, immediate parent last); passing True for the ascending argument will reverse the ordering (immediate parent first, root ancestor last).

If include_self is True, the QuerySet will also include this model instance.

get_children(*args, **kwargs)[source]¶

Returns a QuerySet containing the immediate children of this model instance, in tree order.

The benefit of using this method over the reverse relation provided by the ORM to the instance’s children is that a database query can be avoided in the case where the instance is a leaf node (it has no children).

If called from a template where the tree has been walked by the cache_tree_children filter, no database query is required.
